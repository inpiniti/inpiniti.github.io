# 2 모델링 방법

## 2.1 AAS 상호 작용 파트너 역할 설명

aas : 전체 수명 주기 동안 자산에 대한 정보를 제공

AAS의 서로 다른 상호작용 파트너
- 자산에 대해 서로 다른 견해를 가짐
- AAS에서 자산에 대해 서로 다른 표현을 가질수도 있음

- 이 장에서의 AAS 3가지 역할
    - 해당 자산이 사용될 플랜트를 계획하고 시운전하기 위한 도구 : 상호 작용 가능한 파트너
    - 서비스 요청자 & 서비스 제공자 : 세 번째 상호 작용 파트너 역할

P&P 스테이션의 일부 구성 요소 : 서비스 제공자로 간주될 수 있음

운송장치 및 작업 스테이션과 같은 운송 또는 드릴링 : 서비스 제공자입니다

![AAS 상호 작용 파트너 역할](..\images\06.02 표준화 조사\11.2 aas 참조 모델링\04 AAS 상호 작용 파트너 역할.png)
*AAS 상호 작용 파트너 역할*

### 2.1.1 AAS 상호 작용 파트너 역할 "공장 계획 도구"

구성 요소의 해당 하위 모델 형태로 자산 유형의 AAS속성에 대한 일관되고 상호 운용 가능한 설명이 필요

![플랜트 설계 도구의 하위 모델 선택](..\images\06.02 표준화 조사\11.2 aas 참조 모델링\05 플랜트 설계 도구의 하위 모델 선택.png)
*플랜트 설계 도구의 하위 모델 선택*

- Technical data for the planners of a system (시스템 계획자를 위한 기술 데이터)
    - Electrical, pneumatic, hydraulic and mechanical interfaces (전기, 공압, 유압 및 기계 인터페이스)
    - Operating conditions (e.g. climatic, operating temperatures) (작동 조건(예: climatic, 작동 온도)
    - Functional inputs and outputs (기능 입력 및 출력)
- Assembly instructions (조립 지침)
- Assembly instructions (유지보수 관리)
    
### 2.1.2 AAS 상호 작용 파트너 역할 "시운전 도구"

- AAS
    - 자산의 기술 중립적 상호 운용 가능한 인터페이스를 제공
    - 자산을 테스트 및 시운전을 위한 엔지니어링 및 제어 소프트웨어와 결합 할 수 있음

- 자산 인터페이스로서의 AAS
    - 시운전 엔지니어가 자산을 매개변수화
    - 자산의 기능 테스트를 수행
    - 자산의 기능을 호출하고 제어

- 플랜트에 필요한 프로세스 기능 입증
    - 시운전 엔지니어가 자산 기능을 확인
    - 관련 센서 데이터를 읽을 수 있어야 함
        - 예를 들어 플랜트의 기능에 대한 추가 정보를 제공하는 특성 곡선을 생성하는 데 사용

![시운전 도구의 하위 모델 선택](..\images\06.02 표준화 조사\11.2 aas 참조 모델링\06 시운전 도구의 하위 모델 선택.png)
*시운전 도구의 하위 모델 선택*

- Nameplate (명판)
- Documentations (문서)
- Technical data for commissioning engineers (시운전 엔지니어를 위한 기술 데이터)
    - Measuring and control ranges (측정 및 제어 범위)
    - Electrical and communication interfaces (전기 및 통신)
- Submodels for functional data (기능 데이터의 submodel)
    - Actual data of measured and control values, e.g. "motor current", "position" (측정 및 제어 값 : 예: "모터 전류", "위치")
    - Setting parameters of the devices and components (장치 및 구성 요소의 매개변수 설정)
    - Status and diagnostic values about the plant and about the devices and components (o 설비 및 장치 및 구성 요소에 대한 상태 및 진단 값)

### 2.1.3 AAS 상호 작용 파트너 역할 "서비스 요청자 및 서비스 제공자"

계획자와 시운전 엔지니어를 위해 하위 모델이 제공하는 자산
의 속성과 기능에 대한 의미론적으로 명확한 기계 판독 가능 설
명은 자동화 기술 구성 요소의 상호 운용성과 통합을 높이는
중요한 단계입니다. 

적용 영역은 자산의 실제 생산 능력을 생
산 및 작업 계획에 포함시키고자 하는 운영자의 생산 현장이
며, 따라서 ERP 또는 MES 시스템과의 손쉬운 통합을 목표로
합니다.

- 특정 I4.0 애플리케이션 시나리오
    - 생산 시스템 및 해당 구성 요소의 특정 자율성이 필요합니다(예: 생산 활용의 동적 최적화, 기계 오류로 인한 가동 중지 방지, 주문 기반 생산및 생산 자원의 동적 오케스트레이션). 
    - 로트 크기 1에서 비용 최적화된 생산을 위해

- AAS의 하위 모델
    - 자산 기능 또는 자산이 제공하는 서비스에 대한 정보를 상호 작용 파트너에게 제공할 수 있어야 함

- 시나리오에서의 자산
    - 자율적인 서비스 제공자 또는 서비스 요청자가 됨
    - AAS를 통해 VDI/VDE 2193 - 입찰 절차 지침에 정의된 것과 같은 수평적 프로토콜 기반 상호 작용에 참여 가능
